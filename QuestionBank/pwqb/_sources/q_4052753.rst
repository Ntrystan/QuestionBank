       <p><strong>2017_Q31:</strong></p><p>Consider the following structure definition that represents a time (consisting of an hour and minute value, like 4:30).</p><p>&nbsp;</p><p><span style="font-family: courier new, courier, monospace;">typedef struct {</span><br /><span style="font-family: courier new, courier, monospace;">&nbsp;&nbsp;&nbsp; int hour;</span><br /><span style="font-family: courier new, courier, monospace;">&nbsp; &nbsp; int minutes;</span><br /><span style="font-family: courier new, courier, monospace;">} Time;</span></p><p>&nbsp;</p><p>The following function called IsEarlier() is passed two Time structures as inputs (t1 and t2). The function should return <em>true</em> if the time represented by the first input (t1) <strong>is earlier than</strong> the time represented by the second input (t2). In all other cases, the function should return <em>false</em>. For example, 4:30 <em>is earlier</em> than 6:45, but 4:30 <em>is not earlier</em> than 4:30.</p><p>&nbsp;</p><p><span style="font-family: courier new, courier, monospace;">int IsEarlier(Time t1, Time t2)</span><br /><span style="font-family: courier new, courier, monospace;">{</span><br /><span style="font-family: courier new, courier, monospace;">&nbsp;&nbsp;&nbsp; if (t1.hour &lt; t2.hour) {</span><br /><span style="font-family: courier new, courier, monospace;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; return 1;</span><br /><span style="font-family: courier new, courier, monospace;">&nbsp;&nbsp;&nbsp; } else {</span><br /><span style="font-family: courier new, courier, monospace;">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; return t1.minutes &lt; t2.minutes;</span><br /><span style="font-family: courier new, courier, monospace;">&nbsp;&nbsp;&nbsp; }</span><br /><span style="font-family: courier new, courier, monospace;">}</span></p><p>&nbsp;</p><p><strong>You may have noticed that there is a bug in this function definition</strong>.
